The four schemas a deployer's installation produces and consumes, published under CC-BY 4.0 as JSON Schema (Draft 2020-12). Each schema is downloadable, version-stamped, and linked from the relevant section of the Standard.

Schemas published here implement the Standard's Section 5 artifact definitions verbatim. A Charter authored using a different schema may still align with the Standard if the substantive structure conforms; the published schemas are the authoritative reference shape.

The four schemas

Schema 1

Charter

Version 1.0 · §3 in the Standard

The Charter schema names a single executive seat's decision authority, the artifact set the seat produces, and the dispatch grammar (Mode 1 / Mode 2). The Charter is the load-bearing artifact in the Standard.

Download charter.schema.json
Preview structure 
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://decisionprovenance.ai/schemas/charter/v1.0",
  "type": "object",
  "required": ["charter_id", "seat", "decision_authority", "artifact_set", "dispatch_grammar"],
  "properties": { "...": "..." }
}
Schema 2

Decision Record

Version 1.0 · §4 in the Standard

The Decision Record schema captures a single decision the Charter governs: mode, authorship, the inputs considered, the alternatives evaluated, and the rationale. Records are produced contemporaneously at Level 3.

Download decision-record.schema.json

{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://decisionprovenance.ai/schemas/decision-record/v1.0",
  "type": "object",
  "required": ["record_id", "charter_id", "mode", "authored_by", "decided_at", "rationale"],
  "properties": { "...": "..." }
}
Schema 3

Article 50 Disclosure Metadata

Version 1.0 · §7 in the Standard

The disclosure metadata schema names the AI authorship participation in a decision in a form aligned with the EU AI Act's Article 50 disclosure expectations. Member-State implementations vary; this schema is a structural input to substantive disclosure work, not a substitute for it.

Download article-50-disclosure.schema.json
Preview structure 
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "$id": "https://decisionprovenance.ai/schemas/article-50-disclosure/v1.0",
  "type": "object",
  "required": ["disclosure_id", "decision_record_ref", "ai_participation"],
  "properties": { "...": "..." }
}
Schema 4

Layer 4 Attestation Card

Version 1.0 · §8 in the Standard

The Layer 4 attestation card is the payload of the Reporter API. It carries the self-declared Conformance Level signal and the period over which the signal applies.

  "properties": { "...": "..." }

Schema versioning

Schemas track the Standard's major versions. The current version is v1.0, matching Standard v1.0. When the Standard updates a schema's substantive structure, the schema's $id URL increments accordingly; older schema versions remain available at their historical URLs for citation continuity.
